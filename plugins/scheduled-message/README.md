# 定时消息插件测试指南

## 版本信息
- **版本**: 2.0.0
- **更新日期**: 2026年3月30日
- **主要变更**: 使用数据库 API 替代 KV 存储

## 新功能

### 1. 数据库存储
- 使用 `plugin_scheduled_message_tasks` 表存储定时消息
- 支持数据迁移和版本管理
- 提供更好的性能和查询能力

### 2. 回退机制
- 如果数据库不可用，自动回退到 KV 存储
- 确保插件在各种环境下都能正常工作

## 测试步骤

### 前置条件
1. 确保 CC 项目已启动
2. 确保已登录用户账户
3. 确保插件已安装并启用

### 测试场景

#### 测试 1: 创建定时消息（一次性）
1. 进入聊天页面，选择一个联系人
2. 点击输入区域的"定时"按钮
3. 输入消息内容
4. 选择发送时间（当前时间后 1 分钟）
5. 重复设置选择"不重复"
6. 点击"创建定时消息"
7. **预期结果**: 显示"定时消息创建成功"提示

#### 测试 2: 查看定时消息列表
1. 在创建定时消息弹窗中点击"查看定时列表"
2. **预期结果**:
   - 显示刚才创建的消息
   - 消息状态为"待发送"
   - 显示预计发送时间

#### 测试 3: 编辑定时消息
1. 在定时消息列表中点击消息的"编辑"按钮
2. 修改消息内容或发送时间
3. 点击"保存修改"
4. **预期结果**: 显示"定时消息已更新"提示

#### 测试 4: 取消定时消息
1. 在定时消息列表中点击"取消"按钮
2. 在确认弹窗中点击"确定"
3. **预期结果**:
   - 显示"定时消息已取消"提示
   - 消息从待发送列表中消失

#### 测试 5: 周期重复消息
1. 创建新的定时消息
2. 重复设置选择"每天"
3. 设置截止时间（7 天后）
4. 点击"创建定时消息"
5. **预期结果**: 消息显示"每日"标签

#### 测试 6: 自动发送测试
1. 创建一个定时消息，发送时间设置为 1 分钟后
2. 等待 1 分钟
3. **预期结果**:
   - 消息自动发送
   - 聊天窗口中显示消息
   - 消息状态变为"已发送"

#### 测试 7: 数据库表验证
打开浏览器开发者工具，执行以下 SQL 查询：

```sql
-- 查看所有定时消息
SELECT * FROM plugin_scheduled_message_tasks ORDER BY created_time DESC;

-- 查看待发送的消息
SELECT * FROM plugin_scheduled_message_tasks WHERE status = 0;

-- 查看已发送的消息
SELECT * FROM plugin_scheduled_message_tasks WHERE status = 1;
```

**预期结果**: 查询结果与插件中显示的数据一致

#### 测试 8: 回退机制测试
1. 暂时移除 `database` 权限（修改 manifest.json）
2. 重新加载插件
3. 创建定时消息
4. **预期结果**: 插件使用 KV 存储，功能正常

## 数据库表结构

```sql
CREATE TABLE plugin_scheduled_message_tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    message TEXT NOT NULL,
    contact_type INTEGER NOT NULL,  -- 1=用户, 2=群组
    xxid TEXT NOT NULL,             -- 联系人 ID
    ceid INTEGER,                  -- 联系人元素 ID
    scheduled_time DATETIME NOT NULL,
    status INTEGER DEFAULT 0,      -- 0=待发送, 1=已发送, 2=已取消
    repeat_type TEXT,              -- none, daily, weekly, monthly
    repeat_until DATETIME,         -- 重复截止时间
    sent_time DATETIME,            -- 实际发送时间
    contact_name TEXT,             -- 联系人名称
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_time DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 常见问题

### Q: 插件无法创建定时消息
**A**: 检查以下几点：
1. 是否有 `database` 权限
2. 数据库是否正常初始化
3. 浏览器控制台是否有错误信息

### Q: 定时消息没有自动发送
**A**: 检查以下几点：
1. 插件是否已激活
2. 定时检查是否正常运行
3. 消息状态是否正确

### Q: 数据迁移失败
**A**: 检查以下几点：
1. 数据库连接是否正常
2. 迁移脚本是否正确
3. 查看浏览器控制台错误信息
